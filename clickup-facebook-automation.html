clickup to facebook ad push, end to end

this doc shows the real pipeline used in the product. it is a single trigger in clickup that drives a fully automated ad push in facebook, with strict idempotency and a feedback loop back to clickup.

trigger status = “push to facebook”
job clickup_ad_push
fb api adset + flexible ads
feedback comments + status update

plain message for a friend

i set up clickup so when a task moves to “push to facebook”, it fires a webhook to our backend.

the backend queues a job. the worker pulls the task, validates fields, downloads assets from google drive, creates or reuses the adset, uploads assets, and creates the ads in facebook.

then it posts a summary comment back to clickup and sets the task status to published or failed to push.

retries are safe because we lock per task and reuse ids.

trigger rules

event: taskStatusUpdated only enqueue when status == “push to facebook” always update stored clickup_status skip duplicate webhooks by webhook_id + history_item_id

required clickup fields

primary text 1 headline 1 url destination final submission link (google drive) campaign name (unless test mode) page name

worker steps

fetch task parse custom fields validate inputs resolve campaign + page id download assets from drive create or reuse adset upload assets + create flexible ads comment + status update

idempotency

clickup_push_runs run lock per task clickup_push_items per-variant state reuse adset id via fb_adset_id field reuse assets when hashes match

clickup feedback loop

comments posted: start, download, adset, upload, success, retry, failure success → status published human fix → status failed to push transient error → auto retry, status unchanged

settings knobs

max_concurrent_jobs (default 1) inter_job_delay_seconds (default 2) default_cta, default_ad_status test_mode_enabled, test_campaign_id

api endpoints

method path purpose
post /api/webhooks/clickup receive clickup webhook
get /api/webhooks/clickup/status check webhook config
post /api/webhooks/clickup/setup create/update webhook
get /api/settings/clickup get settings
put /api/settings/clickup update settings

notes

ads are created in flexible format (asset feed spec) google drive downloads are parallelized if any asset download fails, the job fails (no partial push) worker timeout is 90 minutes, stale jobs are cleaned on startup
